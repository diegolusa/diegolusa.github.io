    M 1

    TypeScript é um superset do JavaScript — ou seja, uma linguagem que estende o JavaScript com tipagem estática e recursos avançados de orientação a objetos. A documentação oficial da linguagem encontra-se no endereço https://www.typescriptlang.org/.

    Criado pela Microsoft, ele é transpilado para JavaScript, o que significa que o código TypeScript é convertido em JavaScript antes de ser executado no navegador ou no Node.js. O processo de transpilação é necessário porque os motores de JavaScript não interpretam nativamente TypeScript. Logo, é preciso desta etapa de conversão, de uma linguagem para outra.

    O JavaScript, como sabemos, é uma das três tecnologias que são pilares da Web (HTML e CSS3). Quando surgiu, tinha por objetivo oferecer alguma interatividade às páginas, o que, na maioria das vezes, significava adicionar algumas poucas linhas de código JavaScript ao documento. Contudo, a tecnologia passou a ser muito utilizada e os navegadores precisaram oferecer melhores ferramentas e novas APIs. Com isso, o uso do JavaScript passou a ser cada vez maior no frontend, migrando para a construção de aplicações complexas, com milhares de linhas de código, todas rodando através do motor de execução do navegador.

    Ainda, em certo momento da história, o JavaScript deixou de ser executado exclusivamente no lado cliente e passou a ser possível construir aplicações backend com ele (NodeJS). Nesta perspectiva run everywhere, o JS tornou-se uma das tecnologias mais utilizadas no mundo (TIOBE Index).

    Logo, o TypeScript, como praticamente tudo na tecnologia, nasceu das dificuldades de se manter e desenvolver aplicações JavaScript complexas neste cenário. A natureza dinâmica da linguagem JavaScript, suas variações sintáticas pouco convidativas e a falta de suporte das IDEs para os desenvolvedores foram alguns dos principais motivadores para a criação do TypeScript.

    Vantagem Descrição
    ✅ Tipagem estática Detecta erros durante o desenvolvimento, não apenas em tempo de execução.
    🔍 Autocompletar inteligente Melhora a produtividade em IDEs como VSCode.
    🧠 Melhor manutenção Código mais previsível e seguro.
    🚀 Compatibilidade total Todo código JavaScript válido é também válido em TypeScript.

    Primeiros Passos¶

    Para usar o TypeScript, é necessário ter o Node.js instalado na máquina. Podemos verificar se ele está disponível executando o comando node --version no terminal. Espera-se que o comando apresente a versão do Node disponível.

    Por meio da instalação do Node temos vários programas utilitários a disposição. Um deles é o npm (Node Package Manager), o qual usaremos para instalar pacotes de software e executar outras ferramentas/aplicações. O TypeScript é um pacote, que deverá ser instalado na máquina através do comando npm install -g typescript.

    Em nossos estudos, iremos utilizar o VS Code como IDE. Desta forma, considere que estaremos utilizando o mesmo em todas as nossas atividades de codificação.

    Conhecendo a linguagem¶

    Em seu computador, adicione um novo diretório para armazenar os arquivos do nosso projeto em TypeScript. O nome sugerido é conhecendo-typescript. Após, abra a pasta pelo VS Code (opção Abrir Pasta do menu Arquivo).

    No VSCode, abra um terminal e execute o comando npm init -y. Isso irá adicionar os arquivos necessários para um projeto JavaScript. Na sequência, vamos executar npx tsc --init para criar o arquivo tsconfig.json, arquivo no qual ficam as configurações do transpilador do TypeScript.

    Agora, nosso próximo passo é criar um arquivo para explorarmos a linguagem TypeScript. Vamos chamar o arquivo de basico.ts. Neste arquivo, vamos adicionar o seguinte trecho de código:

    let produto: string = 'Livro';
let preco: number = 29.90;
let desconto: number = 0.1; 

let precoComDesconto: number = preco * (1 - desconto);

console.log(`Produto: ${produto}`);
console.log(`Preço original: R$${preco.toFixed(2)}`);
console.log(`Desconto: ${(desconto * 100).toFixed(0)}%`);
console.log(`Preço com desconto: R$${precoComDesconto.toFixed(2)}`);

function calcularPrecoComDesconto(preco: number, desconto: number): number {
    return preco * (1 - desconto);
}

let novoPrecoComDesconto = calcularPrecoComDesconto(50, 0.2);
console.log(`Novo preço com desconto: R$${novoPrecoComDesconto.toFixed(2)}`);

    Observe as declarações de variáveis acompanham os tipos. Isso é obrigatório, uma vez que o TypeScript impõe tipagem estática. Por meio desta tipagem, será possível evitar diversos erros de codificação típicos de linguagens fracamente tipadas.

    O próximo passo é transpilar o código que criamos para JavaScript, através do comando tsc basico.ts. O resultado do processo será um arquivo de mesmo nome, com extensão JavaScript. Este é o script que podemos executar através do comando node basico.js.

    Tipos básicos¶

    O TypeScript adiciona tipagem explícita às variáveis. Os tipos mais comuns são string, number, booleane suas variações em arrays.

    let nome: string = "Maria";
let idade: number = 25;
let ativo: boolean = true;
let notas: number[] = [8.5, 7.0, 9.2];
let pessoa: [string, number] = ["Carlos", 30]; // Tupla

    Além deles, temos alguns tipos especiais, como undefined, null, any e unknown.

    let indefinido: undefined = undefined;
let nulo: null = null;
let qualquerCoisa: any = "texto"; // evita checagem de tipo
let valorDesconhecido: unknown = 10;

    ⚠️ Atenção: o tipo any deve ser usado com cautela — ele desativa a checagem de tipo.

    Tipagem em funções¶

    Assim como variáveis, parâmetros e retornos de funções devem ser tipados. Tal recurso facilita muito o uso de funções, pois é possível saber antecipadamente os tipos esperados e possíveis necessidades de coerção.

    function somar(a: number, b: number): number {
  return a + b;
}

console.log(somar(5, 3));

    Parâmetros podem ser definidos com valores-padrão, o que os torna opcionais no momento da chamada da função. 

    function saudacao(nome: string, saudacao: string = "Olá"): string {
  return `${saudacao}, ${nome}!`;
}

console.log(saudacao("Ana"));
console.log(saudacao("João", "Bem-vindo"));

    O mesmo vale para funções anônimas e arrow functions. 

    const dobro = (n: number): number => n * 2;
console.log(dobro(4)); // 8

    Interfaces e objetos¶

    Interfaces permitem definir a estrutura de um objeto. 

    interface Usuario {
  nome: string;
  email: string;
  idade?: number; // opcional
}

const usuario1: Usuario = {
  nome: "Paula",
  email: "paula@email.com"
};

    💬 Interfaces garantem que o objeto siga um formato específico.

    6. 🧬 Classes e orientação a objetos¶

    O TypeScript suporta classes, herança e modificadores de acesso.

    class Pessoa {
  nome: string;
  idade: number;

  constructor(nome: string, idade: number) {
    this.nome = nome;
    this.idade = idade;
  }

  apresentar(): string {
    return `Olá, meu nome é ${this.nome} e tenho ${this.idade} anos.`;
  }
}

const maria = new Pessoa("Maria", 28);
console.log(maria.apresentar());

    6.1 Herança¶

    class Funcionario extends Pessoa {
  cargo: string;

  constructor(nome: string, idade: number, cargo: string) {
    super(nome, idade);
    this.cargo = cargo;
  }

  apresentar(): string {
    return `${super.apresentar()} Trabalho como ${this.cargo}.`;
  }
}

const joao = new Funcionario("João", 35, "Desenvolvedor");
console.log(joao.apresentar());

    7. 🧩 Generics (Tipos genéricos)¶

    Generics permitem criar funções ou classes que funcionam com vários tipos de dados.

    function identidade<T>(valor: T): T {
  return valor;
}

console.log(identidade<number>(42));      // 42
console.log(identidade<string>("texto")); // "texto"

    💡 Os genéricos tornam o código mais reutilizável e seguro.

    8. 🧠 Enums e tipos personalizados¶

    8.1 Enum¶

    enum Cor {
  Vermelho = "Vermelho",
  Verde = "Verde",
  Azul = "Azul"
}

let corFavorita: Cor = Cor.Azul;
console.log(corFavorita); // "Azul"

    8.2 Union Types (união de tipos)¶

    let identificador: string | number;
identificador = 101;
identificador = "A102";

    9. ⚡ Trabalhando com módulos¶

    9.1 Exportando e importando¶

    // arquivo mathUtils.ts
export function somar(a: number, b: number): number {
  return a + b;
}

// arquivo app.ts
import { somar } from './mathUtils';
console.log(somar(2, 3));

    💬 Isso facilita a organização e reutilização de código em projetos grandes.

    10. 🧰 Tipos utilitários do TypeScript¶

    TypeScript oferece tipos prontos para simplificar o trabalho.

    Tipo Descrição Exemplo
    Partial<T> Torna todas as propriedades opcionais Partial<Usuario>
    Readonly<T> Impede alterações após a criação Readonly<Usuario>
    Pick<T, K> Escolhe campos específicos Pick<Usuario, "nome" | "email">

    ✅ Conclusão¶

    O TypeScript é hoje uma das linguagens mais utilizadas em grandes aplicações front-end e back-end.
    Ele combina a flexibilidade do JavaScript com a segurança da tipagem estática, tornando o desenvolvimento mais confiável e produtivo.
